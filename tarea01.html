¿Qué es un Protocolo HTTP?

Sus siglas HTTP derivan del inglés HyperText Transfer Protocol o Protocolo de Transferencia de Hiper Textos, el cual nos permite realizar una petición de datos y recursos, como pueden ser documentos en html. El protocolo HTTP funciona en una esquema de petición-respuesta entre el servidor web y el usuario que realiza la petición de transmición de datos.

¿Qué es un Query String?

Son cadenas de consultas que se utilizan para hacer referencia a una interacción con una base de datos. Los Query String permiten acceder a páginas web dinámicas con diferentes variables, es decir, que se utilizan para evitar crear muchos directorios con diferentes variables y megadirectorios de urls.

¿Qué es un Verbo HTTP?

Los verbos HTTP son especificaciones que nos ayudan a indicar acciones sobre el servidor, éstos son GET, POST, PUT, PATCH, DELETE, HEAD, CONNECT, OPTIONS y TRACE. Cada uno indica una acción diferente a la que el servidor debe responder.

¿Qué es un DNS?

DNS es una base de datos distribuida cuya función es traducir los nombres de dominio en las redes. Es decir, que se encargan de traducir la solicitud de un nombre del dominio, por ejemplo, midominio.com/; en una IP específica.

Así, el DNS sirve para traducir los nombres en direcciones IP y viceversa, haciendo posible que podamos navegar por Internet con facilidad, solo teniendo que introducir el nombre de una página o sitio web para acceder a él.

¿Qué es un CDN?

Sus siglas derivan del inglés Content Delivery Network, es un grupo de servidores distribuidos en diferentes ubicaciones geográficas de todo el mundo para permitir la entrega rápida del contenido de un sitio web. También se conoce como red de distribución de contenidos.

Para conseguirlo, un sistema de caché almacena copias temporales de los archivos del sitio web en el servidor proxy más cercano al usuario, acelerando el tiempo de carga de la página.
